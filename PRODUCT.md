# Product

## Register

product

## Users

Developers reviewing changes and coordinating work across Git worktrees, terminals, browsers, files, and coding agents. They need to keep context visible while moving quickly between related tasks.

## Product Purpose

Impala is a desktop workspace for reviewing Git worktree changes and acting on them without leaving the app. Success means users can understand a change, collaborate with agents, and run or inspect the project with minimal navigation overhead.

## Brand Personality

Native, compact, and calm. The interface should feel immediately familiar to users of high-quality developer tools and should stay out of the way of their work.

## Anti-references

Avoid cramped low-contrast controls, tiny labels, ornamental dashboard styling, and unfamiliar interaction patterns. Do not trade readability for density or make secondary chrome compete with the working content.

## Design Principles

- Keep the user's working context visible.
- Make the active location unmistakable.
- Prefer familiar developer-tool interactions and keyboard-first workflows.
- Use density purposefully without sacrificing readable labels or reliable hit targets.
- Let content dominate the interface chrome.

## Accessibility & Inclusion

No additional compliance target is specified. Preserve visible keyboard focus, theme-aware contrast, reduced-motion-friendly state changes, and non-color cues for selected state.
