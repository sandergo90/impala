# Differ Monorepo — Design Spec

Convert the Differ project from a single flat project into a Turborepo monorepo, separating the React frontend from the Tauri/Rust backend.

## Approach

The current repo root (`/Users/sander/Projects/differ/differ/`) becomes the monorepo root. Files are reorganized in place — `.git` stays where it is, preserving full history. The parent directory (`/Users/sander/Projects/differ/`) is NOT the monorepo root.

## Structure

```

├── turbo.json
├── package.json              # Workspace root (bun workspaces)
├── .gitignore
├── docs/
├── .claude/
├── apps/
│   └── desktop/              # React frontend
│       ├── package.json
│       ├── index.html
│       ├── vite.config.ts
│       ├── tsconfig.json
│       ├── tsconfig.node.json
│       ├── components.json
│       ├── public/
│       └── src/
│           ├── main.tsx
│           ├── App.tsx
│           ├── store.ts

│           ├── index.css
│           ├── vite-env.d.ts
│           ├── components/
│           ├── providers/
│           └── lib/
└── backend/
    └── tauri/                # Rust Tauri backend
        ├── Cargo.toml
        ├── Cargo.lock
        ├── tauri.conf.json
        ├── build.rs
        ├── capabilities/
        ├── icons/

```

**Notes:**
- `backend/tauri/gen/` is generated by Tauri — gitignored, will be regenerated
- `backend/tauri/target/` is Cargo build output — gitignored
- `.claude/` and `.superpowers/` stay at monorepo root
- `app-icon.png` moves to `backend/tauri/icons/`

## Root Configuration

### `package.json`

```json
{
  "name": "differ",
  "private": true,
  "workspaces": ["apps/*"],
  "scripts": {
    "dev": "turbo dev",
    "build": "turbo build"
  },
  "devDependencies": {
    "turbo": "<exact version>"
  }
}
```

### `turbo.json`

```json
{
  "$schema": "https://turbo.build/schema.json",
  "tasks": {
    "dev": {
      "persistent": true,
      "cache": false
    },
    "build": {
      "outputs": ["dist/**"],
      "dependsOn": ["^build"]
    },
    "typecheck": {}
  }
}
```

### `.gitignore` updates

Replace `src-tauri/target/` and `src-tauri/gen/` with:
```
backend/tauri/target/
backend/tauri/gen/
```

## Frontend: `apps/desktop/`

Contains all React/Vite/Tailwind/shadcn code.

### `apps/desktop/package.json`

All current dependencies with exact versions (no `^` or `~`). Must include `"type": "module"`.

```json
{
  "name": "@differ/desktop",
  "private": true,
  "type": "module",
  "scripts": {
    "dev": "vite",
    "build": "tsc && vite build",
    "typecheck": "tsc --noEmit",
    "tauri": "tauri"
  },
  "dependencies": { "..." },
  "devDependencies": {
    "@tauri-apps/cli": "<exact version>",
    "..."
  }
}
```

### `vite.config.ts` changes

Remove `**/src-tauri/**` from `watch.ignored` — the Rust backend is now outside the Vite root, so no ignore pattern is needed.

### `components.json`

No changes needed — paths are relative to `components.json` location and will still resolve correctly.

## Backend: `backend/tauri/`

Contains all Rust/Tauri code currently in `src-tauri/`.

### `tauri.conf.json` changes

```json
{
  "build": {
    "devUrl": "http://localhost:1420",
    "frontendDist": "../../apps/desktop/dist",
    "beforeDevCommand": "cd ../../apps/desktop && bun run dev",
    "beforeBuildCommand": "cd ../../apps/desktop && bun run build"
  }
}
```

**Key:** `beforeDevCommand` and `beforeBuildCommand` use `cd ../../apps/desktop` to run from the frontend directory.

### Tauri CLI invocation

The `tauri` script in `apps/desktop/package.json` is the entry point. Tauri v2 looks for `tauri.conf.json` relative to the Cargo project. Since the Cargo project is at `backend/tauri/`, we configure via the Tauri config's `tauri.conf.json` path.

Run from monorepo root:
```bash
cd backend/tauri && cargo tauri dev
```

Or from `apps/desktop/`:
```bash
bunx tauri dev --config ../../backend/tauri/tauri.conf.json
```

The recommended workflow: run `cargo tauri dev` from `backend/tauri/` since that's where `Cargo.toml` lives. The `beforeDevCommand` in the config handles starting the Vite dev server.

## Dependency Versions

All npm dependencies use exact versions — no `^` or `~` prefixes. When writing `package.json` files, strip all version prefixes from current dependencies.

## Migration Steps

All steps happen within the existing git repo (`.git` stays in place):

1. Create `apps/desktop/` directory
2. Move frontend files into `apps/desktop/`: `src/`, `index.html`, `vite.config.ts`, `tsconfig.json`, `tsconfig.node.json`, `components.json`, `public/`
3. Create `backend/tauri/` directory
4. Move `src-tauri/` contents into `backend/tauri/`
5. Move `app-icon.png` into `backend/tauri/icons/`
6. Create root `package.json` with workspaces config
7. Create `apps/desktop/package.json` with exact dependency versions
8. Create `turbo.json`
9. Update `tauri.conf.json`: set `frontendDist`, `beforeDevCommand`, `beforeBuildCommand`
10. Update `vite.config.ts`: remove `src-tauri` from watch ignore
11. Update `.gitignore`: fix `src-tauri/` paths to `backend/tauri/`
12. Remove old root-level files that moved (`package.json` is replaced, old `bun.lock` removed)
13. Run `bun install` from monorepo root
14. Verify: `cd apps/desktop && npx tsc --noEmit`
15. Verify: `cd backend/tauri && cargo check`
16. Verify: `cd backend/tauri && cargo tauri dev` launches the app
17. Commit

## What Stays at Monorepo Root

- `.git/`
- `.gitignore`
- `.claude/`
- `.superpowers/`
- `turbo.json`
- `package.json`
- `bun.lock`
- `node_modules/`
- `docs/`

## What Does NOT Move

- `backend/tauri/` is NOT an npm workspace — it's a Cargo project. Only `apps/*` are in the `workspaces` array.
